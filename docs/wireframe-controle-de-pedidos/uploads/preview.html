1. Fluxos principais

Criar pedido / nota

Início → Nova nota → Cliente → Adicionar itens → Revisar total → Salvar aberta ou Finalizar → Gerar PDF/Compartilhar.

Gerenciar serviços

Serviços → Buscar/filtrar → Novo serviço → Editar preço/status → Confirmar inativação quando destrutivo.

Acompanhar operação

Dashboard → Indicadores do dia → Notas recentes → Filtros por status/período → Detalhe da nota.

Money: text-lg font-bold text-blue-600
Divider: border-t border-slate-100 pt-4
Danger action: text-red-600 ou bg-red-600
Success message: border-green-200 bg-green-50 text-green-700

Observação de arquitetura visual

Para telas com listas, use FlatList como container principal. Para formulários e detalhes, use SafeAreaView + ScrollView. Mantenha ações principais no rodapé quando a tela tiver revisão/salvamento.
