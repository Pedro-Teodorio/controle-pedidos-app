Design System

Controle de Pedidos

Guia visual para um app mobile moderno, intuitivo e minimalista usando React Native, NativeWind e componentes padrão do React Native.

Stack visual: NativeWind + React Native

1. Princípios visuais

Clareza

Informações importantes sempre visíveis: cliente, status, total e número da nota.

Rapidez

Ações principais com poucos toques: criar, salvar, finalizar, gerar PDF e compartilhar.

Confiança

Visual limpo, textos objetivos e feedback claro em ações importantes.

Minimalismo

Poucas cores, espaçamento generoso e componentes consistentes.

7. Classes padrão por tela

Regra importante

Evite colocar FlatList dentro de ScrollView. Isso pode quebrar virtualização, gerar warnings e causar problemas de performance. Use um container para telas comuns e outro para telas de lista.

ScreenContainer — telas com conteúdo simples

Use quando a tela tiver formulário, cards fixos ou conteúdo pequeno. Exemplo: novo serviço, editar serviço, ajustes e detalhe da nota. 

<SafeAreaView className="flex-1 bg-slate-50">
  <ScrollView
    className="flex-1"
    contentContainerClassName="px-4 py-6"
    keyboardShouldPersistTaps="handled"
  >
    {children}
  </ScrollView>
</SafeAreaView>

ListScreenContainer — telas com FlatList

Use quando a tela principal for uma lista. A própria FlatList deve controlar o scroll da tela. 

<SafeAreaView className="flex-1 bg-slate-50">
  <FlatList
    className="flex-1"
    contentContainerClassName="gap-3 px-4 py-6"
    data={data}
    renderItem={({ item }) => <ServicoCard servico={item} />}
    ListHeaderComponent={<ServicosHeader />}
    ListEmptyComponent={<EmptyState title="Nenhum serviço cadastrado" />}
    keyboardShouldPersistTaps="handled"
  />
</SafeAreaView>

9. Regras finais de UI

  • Use bg-slate-50 como fundo padrão das telas.
  • Use bg-white rounded-2xl border border-slate-200 para cards.
  • Botões principais sempre com bg-blue-600 text-white.
  • Ações destrutivas sempre em vermelho e com confirmação.
  • Valores totais devem usar text-blue-600 font-bold.
  • Status devem usar badges semânticos: aberta, finalizada e cancelada.
  • Evite telas densas: prefira espaçamento gap-4 e py-6.
  • Componentes de domínio devem reutilizar os componentes de shared/ui.
